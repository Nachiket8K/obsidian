# Verification and validation

## Upward links
- [[The Delft Method for System Dynamics - Source]]

## Summary
Verification checks whether a model is coded correctly, and validation checks whether it is fit for purpose. Together they anchor model credibility in both technical correctness and decision relevance.

## Key points
- Verification is about internal consistency and correct implementation.
- Validation is about whether the model is credible for its intended use.
- Both structural and behavioural tests matter in SD practice.
