# Model connectors for exploratory modelling

## Upward links
- [[EMA Workbench Documentation - Source]]

## Summary
Model connectors for exploratory modelling are the interface layer that lets one experimentation framework drive models built in different simulation environments. In this vault they explain how DMDU-oriented experiments can span System Dynamics, spreadsheet models, NetLogo, Simio, and related model classes.

## Key points
- Connectors let one exploratory workflow operate across multiple modeling paradigms instead of forcing a single software stack.
- They preserve model-specific implementations while standardizing experimentation and analysis.
- They make it easier to compare policy behaviour across simulation environments and abstraction levels.
