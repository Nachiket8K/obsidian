# Modelling cycle

## Upward links
- [[The Delft Method for System Dynamics - Source]]

## Summary
The modelling cycle is the iterative SD process linking problem articulation, conceptualisation, formulation, evaluation, and policy testing. It treats modelling as a learning loop rather than a linear pipeline.

## Downward links
- [[Delft SD Ch 01 - Introduction to System Dynamics]]
- [[Delft SD Ch 02 - The first modelling cycle]]

## Key points
- Each phase informs and revises the others.
- Model development is cyclical, not one-pass.
- The cycle supports both explanation and policy design.
