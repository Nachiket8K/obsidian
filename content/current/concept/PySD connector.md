# PySD connector

## Upward links
- [[EMA Workbench Documentation - Source]]
- [[The Delft Method for System Dynamics - Source]]

## Summary
The PySD connector is the bridge that lets EMA Workbench run executable System Dynamics models inside an exploratory modeling workflow. In this vault it is the clearest practical integration point between the DMDU notebook and the System Dynamics graph.

## Downward links
- [[Python Library Documentation/emaworkbench/ema documentation/connectors/pysd connector|EMA Workbench PySD connector docs]]
- [[Python Library Documentation/emaworkbench/examples/example pysd teacup|EMA Workbench PySD teacup example]]
- [[Python Library Documentation/emaworkbench/getting started/overview|EMA Workbench overview]]

## Key points
- It turns a System Dynamics model into something that can be sampled, stress-tested, and optimized through a shared experimentation framework.
- It is especially useful when DMDU-style uncertainty exploration needs to run over stock-flow style models.
- It helps connect conceptual SD notes to executable computational workflows.
