# Policy testing (system dynamics)

## Upward links
- [[The Delft Method for System Dynamics - Source]]
- [[EMA Workbench Documentation - Source]]

## Summary
The SD phase that uses models to design, compare, and stress-test policies, including adaptive policies, across plausible futures. It overlaps strongly with DMDU because it treats policy robustness as a first-class modelling objective.

## Downward links
- [[Delft SD Ch 09 - Model use and policy testing]]

## Core ideas
- Identify policy levers and test how they change system behaviour.
- Compare static and adaptive policies explicitly.
- Assess policy performance across different scenarios rather than only a base case.
- Treat robustness as a practical policy property, not an abstract ideal.
