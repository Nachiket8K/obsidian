# Problem articulation and conceptualisation

## Upward links
- [[The Delft Method for System Dynamics - Source]]

## Summary
A front-end SD method for defining the problem, bounding the system, sketching structure, and proposing a dynamic hypothesis before quantification begins. It is a useful bridge between SD practice and DMDU-style framing work.

## Downward links
- [[Delft SD Ch 01 - Introduction to System Dynamics]]
- [[Delft SD Ch 06 - Problem Formulation and Conceptualisation]]

## Core ideas
- Formulate the problem rather than trying to model the whole world.
- Use boundaries, time horizon, and reference modes to focus the model.
- Create diagrams and dynamic hypotheses before writing equations.
- Support later analysis by making structural assumptions explicit early.
