# System diagram

## Upward links
- [[Policy Analysis of Multi-Actor Systems - Source]]

## Summary
A system diagram delineates the relevant system, its boundaries, its factors, and their main relationships so a policy problem can be discussed in an explicit structured form.

## Downward links
- [[PAMS Ch 03 - Systems Analysis]]
- [[PAMS Ch 03 S03 - A Method to Develop a System Diagram]]

## Core ideas
- Make system boundaries explicit.
- Show how internal and external factors relate to the problem.
- Provide a shared picture for later synthesis, scenario work, or modeling.
