# Second Brain Constitution
## AI Governance Rules for the Obsidian Knowledge System

Version: 1.0

# Purpose

This vault is a persistent knowledge system designed to support:

- Research
- Simulation and modelling
- Decision support
- Project execution
- Knowledge accumulation
- Method development
- Reusable model development

The vault is the authoritative source of truth.

AI systems operating within this vault are assistants to the knowledge system and must not act as independent authors.

All modifications must preserve traceability, context, and knowledge reuse.

# Core Principles

## Principle 1 — Knowledge is Permanent

Knowledge should never be deleted simply because it appears obsolete.

Instead:
- Archive old knowledge
- Mark superseded knowledge
- Preserve historical context

AI must never destroy information without explicit instruction.

## Principle 2 — Every Note Has a Purpose

Every note must belong to one of the following categories:

- Knowledge
- Methods
- Projects
- Models
- Decisions
- Assets
- Research

AI should classify new notes accordingly.

## Principle 3 — Single Source of Truth

Knowledge should exist in only one canonical location.

Duplicate notes should not be created.

Instead:
- Create links
- Create aliases
- Create references

If a note already exists, AI must enrich it rather than create another copy.

## Principle 4 — Atomic Knowledge

Each note should contain one primary idea.

Large topics must be decomposed.

## Principle 5 — Reusability First

Knowledge should be written for future reuse.

Extract reusable concepts, methods and models from projects and link them back.
