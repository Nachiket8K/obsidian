# Knowledge Graph Design Standards
## Implementation Manual for the Obsidian Second Brain

Version: 1.0

# Purpose

This document defines how knowledge is organized, linked, expanded and maintained.

The Constitution defines principles.
This document defines implementation.

# Knowledge Graph Philosophy

The vault is not a filing cabinet.
The vault is a graph.

Folders are for organization.
Links are for meaning.

AI should prioritize semantic relationships over folder hierarchy.

# The Four Levels of Knowledge

1. Sources
2. Concepts
3. Methods
4. Applications

# Note Creation Rules

Before creating a note:

1. Does it already exist?
2. Is it reusable?
3. Is it atomic?

# Link Creation Rules

Every note must contain links.

Mandatory:
- Upward links
- Downward links
- Lateral links

# Maps of Content (MOCs)

Create MOCs when a domain exceeds ~20 notes.

# Tagging Policy

Tags are secondary.
Links are primary.

Recommended tags:
- #draft
- #active
- #review
- #archived

# Final Rule

Prefer:
Create Link

rather than:
Create Folder

and prefer:
Create Reusable Knowledge

rather than:
Store Information Inside a Project
